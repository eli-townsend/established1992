Border-Left: Required;

Have you ever viewed a website with chicken pox? A form on a web page with numerous red required field asterisks or alternatively labeling fields that are optional when required fields outnumber optional fields. At the company I work for, we stumbled upon this issue in the modernization phase from AS400 to Java Web Applications. As UI/UX designers, we did not want our applications to have chicken pox. Our forms were heavy with several required and optional fields.

We designed a solution by taking advantage of the border-left property in CSS (W3Schools). On input fields that are required (by class name or required boolean value) we placed the border-left property with 3 pixels of solid border instead of using an asterisk.

<input type="text" placeholder="placeholder text" class="required">
<input type="text" placeholder="placeholder text" required>

In CSS, this is achievable by calling the required class selector required or by calling the pseudo class selector :required. To add interactivity and provide the user informative feedback – once the cursor was focused into the input, we changed the border-left property from 3 pixels to 6 pixels.

While this works for most users we wanted to make a note of accessibility about this solution. For example, if a user were to have a form of color blindness, you will need to vary the color so that it may be easily distinguishable from the rest of the input border, background color or any other styles that might be applied to your inputs. We will also put a transition on the initial state of the input so once on focus, there will be 0.3 seconds time to transition from 3 to 6 pixels.

input.required, select.required, textarea.required, input:required, select:required, textarea:required {
border-left:#E96751 solid 3px;
-webkit-transition: border 0.3s;
transition: border 0.3s;
}
input[type=text].required:focus, input[type=date].required:focus, select.required:focus, textarea.required:focus{
border-left:#E96751 solid 6px;
}

If you use certain front-end frameworks, they will showcase errors, warnings, or success states differently, whether that be background color changed, border changed, etc. As a standard, we created this to work with validation so depending on which state the input is in, the border-left will depict that state with green for success or red for error. If the user is already aware of the border-left changing dynamically through validation, it gives users a sense of standard and consistent feedback throughout the process.

See the Pen Border-Left Inputs by Elijah Townsend (@Eli-townsend) on CodePen.

I would love to hear your input on this topic and see what everyone else has done as a solution to the chicken pox.
